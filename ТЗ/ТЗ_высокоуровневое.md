# Техническое задание (ТЗ): Высокоуровневое описание системы

Детальная спецификация: `ТЗ_конкретное.md`.

## 1. Назначение системы
- Система предоставляет мобильный инвестиционный продукт с симуляцией торговли.
- Пользователь получает доступ к портфелю, котировкам, операциям и аналитике.
- Исполнение ордеров выполняется внутри backend-контура.
- Детализация: [введение](ТЗ_конкретное.md#intro), [продуктовые предпосылки](ТЗ_конкретное.md#product-assumptions).

## 2. Продуктовая модель
- Инструменты: акции и облигации.
- Валюта расчётов: RUB.
- Модель торговли: только `Market`, полное исполнение при достаточных средствах/позиции.
- Счета: один инвестиционный счёт + виртуальный денежный счёт.
- Детализация: [продуктовые предпосылки](ТЗ_конкретное.md#product-assumptions), [фиксированные требования](ТЗ_конкретное.md#fixed-requirements).

## 3. архитектура
- Архитектура: микросервисная.
- Ключевые подсистемы:
- Mobile Apps (Android и React Native).
- Backend API (auth, portfolio, orders, operations, analytics, quotes WS).
- Quotes pipeline (driver, reader/collector, Valkey, ClickHouse).
- Детализация: [архитектура](ТЗ_конкретное.md#architecture), [технологический стек](ТЗ_конкретное.md#tech-stack), [биржевой контур](ТЗ_конкретное.md#vtp-pipeline).

## 5. Данные и хранение
- PostgreSQL: транзакционные сущности (пользователи, счета, позиции, ордера, операции).
- ClickHouse: raw ticks и аналитические агрегаты, retention raw ticks = 1 месяц.
- Valkey: Pub/Sub для котировок и событий.
- Детализация: [модель данных](ТЗ_конкретное.md#data-model), [фиксированные требования](ТЗ_конкретное.md#fixed-requirements).

## 6. Внешние контракты
- REST для команд/чтения между клиентом и бэком, WebSocket для realtime котировок, grpc между микросервисами.
- В MVP включены register/login/refresh/logout, deposit/reset, торговые и аналитические endpoints.
- Ограничение подписок WS: до 50 инструментов на соединение.
- Детализация: [API](ТЗ_конкретное.md#api), [минимальные endpoints](ТЗ_конкретное.md#api-minimal).

## 7. Пользовательские сценарии
- Основные экраны: портфель, инструменты, карточка инструмента, профиль, операции, аналитика.
- Торговый сценарий: Market BUY/SELL, проверки достаточности средств/позиции, мгновенное исполнение.
- Правило цены: `LAST`, fallback на mid или последнюю известную цену.
- Детализация: [функциональные требования mobile](ТЗ_конкретное.md#mobile-functional), [экранные требования](ТЗ_конкретное.md#screens).

## 8. Нефункциональные требования
- Нагрузка: до 10 000 одновременных пользователей и WS-соединений.
- Требования к производительности, отказоустойчивости, безопасности и наблюдаемости.
- Детализация: [NFR](ТЗ_конкретное.md#nfr), [нагрузочное тестирование](ТЗ_конкретное.md#load-testing), [тестирование](ТЗ_конкретное.md#testing).

## 9. Поставка и готовность
- План поставки: MVP-1 и последующие расширения.
- Критерии готовности: функциональность, тесты, наблюдаемость, нагрузка, безопасность.
- Открытые вопросы остаются отдельным треком.
- Детализация: [план поставки](ТЗ_конкретное.md#delivery-plan), [открытые вопросы](ТЗ_конкретное.md#open-questions), [DoD](ТЗ_конкретное.md#dod).
